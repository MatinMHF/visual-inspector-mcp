# visual-inspector-mcp

یک سرور [MCP](https://modelcontextprotocol.io) (پروتکل Model Context) که به Claude Code — یا هر کلاینت سازگار با MCP — امکان می‌دهد صفحات وب رندرشده را **به‌صورت بصری بررسی کند**، از طریق یک مرورگر Chromium بی‌سر (headless) پایدار که با [Playwright](https://playwright.dev) کار می‌کند.

به‌جای حدس زدن وضعیت UI صرفاً از روی کد منبع، دستیار هوش مصنوعی شما می‌تواند به یک URL رفته، اسکرین‌شات بگیرد، روی المان‌ها کلیک کند، اندازه viewport را تغییر دهد و لاگ‌های console مرورگر را بخواند — همه در یک سشن صفحه‌ی پایدار.

## چرا این پروژه ساخته شد

بیشتر دستیارهای هوش مصنوعی برای کدنویسی فقط فایل‌های منبع شما را می‌خوانند. `visual-inspector-mcp` به دستیار شما یک پنجره‌ی مرورگر واقعی می‌دهد تا بتواند:

- **ببیند** صفحه واقعاً چه شکلی است (نه فقط آنچه HTML می‌گوید)
- **با UI تعامل کند** — منوها را باز کند، تب‌ها را تغییر دهد، modal‌ها را فعال کند — قبل از گرفتن اسکرین‌شات
- **به‌صورت بصری دیباگ کند** با تطبیق یک layout شکسته با خطاهای console در همان سشن
- **نقاط شکست responsive را بررسی کند** با تغییر لحظه‌ای اندازه viewport

همه این‌ها از طریق یک سرور MCP سبک‌وزن — بدون Docker، بدون سرویس‌های اضافی.

## ابزارها

| ابزار | توضیح | اثرات جانبی؟ |
| --- | --- | --- |
| `navigate` | بارگذاری یک URL در صفحه headless پایدار | اتصالات شبکه باز می‌کند |
| `screenshot` | گرفتن اسکرین‌شات از viewport، صفحه کامل، یا یک CSS selector | فقط خواندنی |
| `click` | کلیک روی یک المان با CSS selector یا Playwright locator | وضعیت صفحه را تغییر می‌دهد |
| `resize` | تغییر اندازه viewport مرورگر | وضعیت صفحه را تغییر می‌دهد |
| `console_logs` | بازگرداندن پیام‌های اخیر console مرورگر و خطاهای صفحه | فقط خواندنی |

هر ابزار با متادیتای `readOnlyHint`، `destructiveHint` و `openWorldHint` حاشیه‌نویسی شده است تا کلاینت‌های MCP بتوانند تأییدیه‌های مناسب را نمایش دهند.

### `navigate`

یک URL را در صفحه مرورگر پایدار بارگذاری می‌کند. صفحه برای فراخوانی‌های بعدی `click`، `screenshot` یا `console_logs` باز می‌ماند.

| پارامتر | نوع | الزامی | توضیحات |
| --- | --- | --- | --- |
| `url` | string | بله | URL برای بارگذاری (از `http://`، `https://`، `file://` و URI های `data:` پشتیبانی می‌کند) |
| `waitForSelector` | string | خیر | قبل از بازگشت منتظر این selector می‌ماند — مفید برای SPAها |

URL حل‌شده و عنوان صفحه را در یک خط برمی‌گرداند: `https://example.com — "Example Domain"`.

### `screenshot`

اسکرین‌شاتی از صفحه جاری گرفته و آن را به‌صورت یک بلاک تصویر base64-encoded برمی‌گرداند.

| پارامتر | نوع | الزامی | توضیحات |
| --- | --- | --- | --- |
| `selector` | string | خیر | فقط این المان را캡쳐 می‌کند (ارزان‌ترین حالت)؛ CSS selector یا Playwright locator |
| `fullPage` | boolean | خیر | صفحه قابل اسکرول کامل را캡쳐 می‌کند (گران‌ترین حالت)؛ پیش‌فرض viewport است |
| `format` | `"png"` یا `"jpeg"` | خیر | `png` (پیش‌فرض، بدون اتلاف) یا `jpeg` (کوچک‌تر؛ مناسب بررسی layout) |
| `quality` | عدد ۱–۱۰۰ | خیر | کیفیت JPEG (پیش‌فرض ۸۰)؛ برای PNG نادیده گرفته می‌شود |

نیاز دارد که ابتدا `navigate` فراخوانی شده باشد.

### `click`

روی یک المان کلیک می‌کند تا به یک وضعیت UI برسد (باز کردن منو، modal یا تب) قبل از گرفتن اسکرین‌شات.

| پارامتر | نوع | الزامی | توضیحات |
| --- | --- | --- | --- |
| `selector` | string | بله | CSS selector یا Playwright locator المانی که باید کلیک شود |

### `resize`

aندازه viewport مرورگر را برای بررسی یک نقطه شکست responsive تغییر می‌دهد.

| پارامتر | نوع | الزامی | توضیحات |
| --- | --- | --- | --- |
| `width` | number | بله | عرض viewport بر حسب پیکسل |
| `height` | number | بله | ارتفاع viewport بر حسب پیکسل |

### `console_logs`

پیام‌های اخیر console مرورگر و خطاهای صفحه را برمی‌گرداند — مفید برای تطبیق یک مشکل بصری با یک خطای JavaScript.

| پارامتر | نوع | الزامی | توضیحات |
| --- | --- | --- | --- |
| `limit` | number | خیر | حداکثر تعداد ورودی‌هایی که برگردانده می‌شود، جدیدترین اول (پیش‌فرض ۳۰) |
| `level` | `"all"` یا `"error"` | خیر | `"all"` (پیش‌فرض) یا فقط خطاها/اخطارها |
| `clear` | boolean | خیر | بافر لاگ را پس از خواندن پاک می‌کند |
| `withTimestamps` | boolean | خیر | timestamp ISO را به ابتدای هر ورودی اضافه می‌کند (پیش‌فرض false) |

## نصب

### پیش‌نیازها

- [Node.js](https://nodejs.org/) نسخه ۱۸ به بالا (برای پشتیبانی بومی از `fetch` و ESM)
- یک ترمینال / shell

### کلون و نصب وابستگی‌ها

```bash
git clone https://github.com/MatinMHF/visual-inspector-mcp.git
cd visual-inspector-mcp
npm install          # همچنین `playwright install chromium` را از طریق postinstall اجرا می‌کند
```

اگر Chromium به‌طور خودکار نصب نشد، اجرا کنید:

```bash
npx playwright install chromium
```

## استفاده

### اجرای مستقیم

```bash
node index.js
```

سرور از طریق **stdio** ارتباط برقرار می‌کند، پس باید توسط یک کلاینت MCP راه‌اندازی شود، نه به‌صورت تعاملی و مستقل.

### ثبت در Claude Code

```bash
claude mcp add visual-inspector -s user -- node /path/to/visual-inspector-mcp/index.js
```

در PowerShell ویندوز، جداکننده `--` را با کوتیشن بنویسید تا PowerShell آن را به‌طور خاموش حذف نکند:

```powershell
claude mcp add visual-inspector -s user "--" node "C:/path/to/visual-inspector-mcp/index.js"
```

### ثبت در Claude Desktop (یا هر کلاینت سازگار با MCP)

یک ورودی به تنظیمات سرور MCP کلاینت خود اضافه کنید (مثلاً `claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "visual-inspector": {
      "command": "node",
      "args": ["C:/path/to/visual-inspector-mcp/index.js"]
    }
  }
}
```

### مثال یک جریان کاری

یک نشست معمول پس از ثبت سرور در کلاینت MCP شما:

1. **«به سرور توسعه من برو»** → فراخوانی `navigate` با `http://localhost:3000`
2. **«یک اسکرین‌شات بگیر»** → فراخوانی `screenshot` برای مشاهده UI رندرشده جاری
3. **«روی منوی hamburger کلیک کن»** → فراخوانی `click` با selector، سپس دوباره `screenshot`
4. **«در عرض موبایل چک کن»** → فراخوانی `resize` با `{ width: 375, height: 667 }`، سپس `screenshot`
5. **«آیا خطای JS داری؟»** → فراخوانی `console_logs` با `{ level: "error" }`

## اجرای تست دود

یک تست دود (smoke test) خودکفا، سرور را به‌عنوان یک subprocess راه‌اندازی می‌کند، از طریق stdio با MCP صحبت می‌کند و همه پنج ابزار را آزمایش می‌کند:

```bash
npm test
```

تست فایل‌های `test-full.png` و `test-element.png` را در دایرکتوری کاری به‌عنوان اثبات کارکرد اسکرین‌شات‌ها می‌نویسد. هر دو فایل را می‌توان پس از آن حذف کرد.

## نکات امنیتی

- سرور یک نمونه Chromium **headless** راه‌اندازی می‌کند؛ این نمونه از طریق URL های `file://` به شبکه محلی و سیستم‌فایل شما دسترسی دارد.
- فقط آن را با کلاینت‌ها و سشن‌های MCP که به آن‌ها اعتماد دارید ثبت کنید.
- `click` با `destructiveHint: true` علامت‌گذاری شده — کلاینت‌های خوش‌رفتار باید قبل از فراخوانی آن در صفحات production تأییدیه بگیرند.
- از آنجا که همه ابزارها `openWorldHint: true` دارند (تماس‌های شبکه واقعی)، بدون درک ریسک، آن را به سرویس‌های داخلی حساس نشانه نگیرید.
